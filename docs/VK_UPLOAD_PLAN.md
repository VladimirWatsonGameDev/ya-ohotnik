# План загрузки в VK Mini Apps

## 1. Подготовить кабинет приложения

1. Создать приложение в кабинете VK для разработчиков.
2. Выбрать тип VK Mini Apps.
3. Заполнить название: `Я Охотник`.
4. Зафиксировать ID приложения: `54611895`, внутренний адрес VK: `app54611895`.
5. Добавить краткое описание, полное описание и слоган.
6. Загрузить иконки, обложки, скриншоты, возрастной рейтинг и юридические ссылки.
7. Указать production URL, на котором будет доступен `index.html` из сборки.

## 2. Подготовить сборку

```bash
npm install
npm run build
```

После команды появится папка `dist`. В ней должны быть:

- `index.html`;
- JS/CSS bundle в `assets`;
- оптимизированные изображения;
- статические файлы из `public`.

## 3. Разместить файлы

Варианты размещения:

- статический хостинг с HTTPS;
- сервер разработчика;
- CDN/объектное хранилище;
- внутренний VK-хостинг, если он доступен в кабинете приложения.

Главное условие: production URL должен открывать `index.html`, а все относительные пути должны работать. В `vite.config.ts` уже выставлено `base: "./"`, чтобы сборка не ломалась при вложенном пути.

`https://vk.com/app54611895` - это публичная VK-ссылка на приложение после настройки. Сама игра при этом должна лежать на HTTPS-хостинге, а URL этого хостинга указывается в настройках приложения.

## 4. Проверить VK Bridge

Минимальный чек:

- `VKWebAppInit` вызывается при старте;
- приложение запускается внутри VK webview;
- пользователь корректно определяется через `VKWebAppGetUserInfo`;
- прогресс сохраняется через `VKWebAppStorageSet`;
- прогресс читается через `VKWebAppStorageGet`;
- платежный сценарий открывается только после настройки магазина и серверной проверки.

## 5. Подготовить модерацию

До отправки на проверку:

- убрать тестовые ассеты или пометить билд как тестовый;
- проверить, что игра не требует внешней регистрации;
- добавить политику конфиденциальности;
- добавить пользовательское соглашение;
- для бесплатного MVP не указывать платные покупки в карточке приложения;
- после включения платежей описать платные покупки и отсутствие обязательной оплаты;
- проверить возрастной рейтинг из-за темы охоты и оружия.

## 6. Release checklist

- `npm run build` проходит без ошибок.
- Игра открывается по production URL.
- Первый запуск не зависает на загрузке.
- Все ассеты догружаются.
- Уровень 1 можно пройти.
- Уровень 30 включает ночной режим.
- Бонусные уровни 10, 20, 30 ... 100 запускаются.
- Магазин не списывает монеты при ошибке.
- Платежи проверяются на сервере.
- Рейтинг не принимает поддельные очки с клиента.
