# Монетизация и магазин

## Текущий режим MVP

На первом этапе игра работает бесплатно:

- реальные платежи отключены;
- монеты выдаются через демо-кнопку в магазине;
- все винтовки покупаются только за внутриигровые монеты;
- дополнительных серверных проверок пока нет, потому что нет реальных списаний.

Когда игра будет готова к публикации и тестам удержания, демо-кнопку нужно заменить на VK-платежи и backend-проверку покупок.

## Валюты

### Монеты

Внутриигровая валюта. Нужна для:

- покупки дополнительных патронов;
- покупки винтовок;
- будущих косметических предметов.

### Реальные платежи

Будущий платный режим: пользователь покупает наборы монет через VK платежную форму. Клиент только открывает платежный сценарий. Факт оплаты должен подтверждаться сервером.

## Наборы монет

- `coins_300` - 300 монет;
- `coins_1000` - 1000 монет;
- `coins_2500` - 2500 монет;
- `coins_7000` - 7000 монет.

## Винтовки

- `ИЖ Старт` - базовая, бесплатная.
- `Сокол` - быстрее пуля, чуть стабильнее прицел.
- `Таежник` - хороший баланс скорости и стабильности.
- `Ночной след` - дорогая винтовка для поздних ночных уровней.

## Правило честной экономики

Платные монеты ускоряют прогресс, но не должны быть обязательными для прохождения. У каждого уровня патронов достаточно для количества целей, а магазин помогает компенсировать промахи и повышать комфорт.

## Серверная защита

Для бесплатного MVP серверная защита не обязательна, если рейтинг и покупки не имеют реальной ценности.

Для платной версии нельзя доверять клиенту:

- начисление монет после оплаты;
- запись рекорда в рейтинг;
- покупка оружия;
- выдача ежедневных наград.

Нужен backend endpoint:

```text
POST /api/vk/payment/callback
POST /api/player/sync
POST /api/leaderboard/submit
GET  /api/leaderboard/top
```

Минимальная серверная логика:

1. получить callback от VK;
2. проверить подпись и id заказа;
3. начислить монеты пользователю;
4. вернуть клиенту актуальный баланс;
5. принимать рекорд только если он соответствует пройденному уровню и лимитам.
